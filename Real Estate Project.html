import pandas as pd

# Sample DataFrame (you can replace this with your actual data)

# Define a function to calculate IQR and detect outliers
def detect_outliers_iqr(df, column_name):
    Q1 = df[column_name].quantile(0.25)
    Q3 = df[column_name].quantile(0.75)
    IQR = Q3 - Q1

    lower_bound = Q1 - 1.5 * IQR
    upper_bound = Q3 + 1.5 * IQR

    print(f"\nColumn: {column_name}")
    print(f"Lower Bound: {lower_bound}")
    print(f"Upper Bound: {upper_bound}")

    # Detect outliers
    outliers = df[(df[column_name] < lower_bound) | (df[column_name] > upper_bound)]
    return outliers

# List of columns to check
columns_to_check = ['Price', 'SquareFeet', 'LotSize', 'GarageSpaces']

# Loop through each column and detect outliers
for column in columns_to_check:
    outliers = detect_outliers_iqr(df, column)
    print(f"\nOutliers for {column}:")
    print(outliers)

CONCLUSION:¶

In this project, the goal was to analyze and predict housing prices using various regression models, including Linear Regression, Ridge, Lasso, and Random Forest. Initially,¶

the dataset showed very low correlation between most features and price, indicating weak linear relationships. As a result, early models performed poorly, with negative R² scores and high mean squared errors — highlighting that the raw features alone were insufficient for effective prediction.¶

- To address this, we applied feature engineering techniques, such as log transformations (LogPrice, LogSquareFeet, LogLotSize) and the creation of new variables like HouseAge, PricePerSqFeet, and LargeLot.¶

The most impactful transformation involved replacing the categorical ZipCode with a numerical grouping (ZipValue) based on average prices per zip. This not only preserved location¶

information but also allowed linear models to leverage it without high-dimensional one-hot encoding.¶

- After incorporating these features, model performance improved dramatically. Ridge Regression achieved an R² of 0.9998 with an extremely low error, and Lasso reached 0.9677,¶

highlighting the strength of the engineered features — especially ZipValue, LogSquareFeet, and PricePerSqFeet.¶

Lasso further simplified the model by identifying ZipValue as the dominant predictor, effectively regularizing all other features to zero.¶

-Visualizations confirmed the model’s accuracy and emphasized the strong relationship between ZipValue and LogPrice. Residual analysis showed no major biases or patterns, supporting the model’s reliability.¶

-In conclusion, this analysis demonstrates that smart feature engineering and domain-driven transformation can turn a weak dataset into a powerful predictive model.¶

While raw features provided limited insight, engineered variables — particularly those that capture location, scale, and value density — enabled highly accurate and interpretable results.¶

-This approach can be a valuable framework for future real estate modeling tasks or similar domains where raw correlations may be misleading.¶
